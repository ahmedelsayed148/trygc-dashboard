Build a production-ready internal web platform called “Trygc OPS Command” (also acceptable label: “TryGC Dashboard”)
  for managing campaign operations, team execution, community workflows, coverage tracking, updates, reporting, and
  admin configuration.

  This is not a generic task app. It is a role-based operations command center for a campaign/influencer marketing
  operations team. It must feel like a premium internal dashboard with bold monochrome styling, strong hierarchy,
  rounded cards, fast interactions, admin/member role separation, and full CRUD behavior across all workspace modules.

  Core product goal:
  Create one unified workspace where admins and members can log in, manage campaigns and subtasks, track coverage and
  community operations, log mistakes and wins, organize messy updates into professional summaries, create shift
  handovers, manage shared quick-link widgets, export/import data, and control feature access per user.

  Preferred implementation defaults if no stack is provided:
  - Frontend: React + TypeScript
  - Styling: Tailwind CSS + component primitives
  - Routing: client-side SPA routing
  - Charts: Recharts or equivalent
  - Backend: Supabase (Auth + database + storage + edge/server functions) or equivalent
  - State/data fetching: React Query or equivalent
  - File parsing: XLSX support for imports
  - Must support desktop and mobile
  - Must support light and dark mode
  - Must be modular and maintainable, not a single giant file

  Visual direction:
  - Primary aesthetic: black, white, zinc/gray, minimal but high-contrast
  - Large rounded cards and rounded buttons
  - Bold typography and dashboard polish
  - Subtle gradients and restrained motion
  - Serious operations-tool feel, not playful consumer SaaS
  - No purple-heavy default theme
  - Responsive sidebar on desktop, bottom nav on mobile
  - Top bar with sync state, quick actions, user menu, theme toggle, refresh, and shortcuts
  - Command palette opened by Ctrl/Cmd + K

  Authentication and onboarding:
  - Email/password sign-in and sign-up
  - Persist sessions
  - After login, auto-register or auto-provision user profile if needed
  - Roles:
    - owner
    - admin
    - member
  - Owner has permanent full access and cannot be downgraded by others
  - First-time users must go through an interactive platform demo/walkthrough before entering the workspace
  - After demo completion:
    - admins land on the main dashboard
    - members land on the personal dashboard

  Global workspace behavior:
  - Auto-save changes with debounce
  - Show sync states such as idle, loading, online, syncing, error
  - Keep local fallback storage for selected workspace modules so recent changes remain visible even if backend sync
  fails
  - Merge server and local records by latest updatedAt
  - Refresh workspace data from backend on demand
  - Feature-based navigation: each user only sees enabled modules
  - Some features must always remain available:
    - Coverage
    - Widgets
  - Provide lazy-loaded routes for major screens
  - Provide loading and error states
  - Provide search, filters, date range filters, and empty states on most data-heavy screens
